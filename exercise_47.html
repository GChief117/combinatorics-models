Exercise 47 — Prove $\#(\text{flatten}\;t) = \text{count}\;t$

Induction proof — establishing that flattening a tree preserves its leaf count.

The Scenario

The flatten function converts a tree into a sequence of its leaf values, while count counts the number of leaves. We prove by structural induction on $\text{Tree}$ that for any tree $t$:

$$\#(\text{flatten}\;t) = \text{count}\;t$$

That is, the length of the flattened sequence equals the number of leaves.

Given Definitions

Free Type (from Exercise 46)

$$\text{Tree} ::= \text{stalk} \mid \text{leaf}\langle\!\langle \mathbb{N} \rangle\!\rangle \mid \text{branch}\langle\!\langle \text{Tree} \times \text{Tree} \rangle\!\rangle$$

Definition of count

count.1: $\;\text{count}\;\text{stalk} = 0$

count.2: $\;\forall\, n : \mathbb{N} \bullet \text{count}(\text{leaf}\;n) = 1$

count.3: $\;\forall\, t_1, t_2 : \text{Tree} \bullet \text{count}(\text{branch}(t_1, t_2)) = \text{count}\;t_1 + \text{count}\;t_2$

Definition of flatten

flatten.1: $\;\text{flatten}\;\text{stalk} = \langle\rangle$

flatten.2: $\;\forall\, n : \mathbb{N} \bullet \text{flatten}(\text{leaf}\;n) = \langle n \rangle$

flatten.3: $\;\forall\, t_1, t_2 : \text{Tree} \bullet \text{flatten}(\text{branch}(t_1, t_2)) = \text{flatten}\;t_1 \;\frown\; \text{flatten}\;t_2$

Sequence Length Facts

len.1: $\;\#\langle\rangle = 0$  —  empty sequence has length zero

len.2: $\;\#\langle n \rangle = 1$  —  singleton sequence has length one

len.3: $\;\#(s \frown t) = \#s + \#t$  —  length distributes over concatenation

Induction Principle for Tree

To prove a property $P(t)$ for all trees $t$, we must show three things:

ObligationWhat to Show
Base case 1$P(\text{stalk})$
Base case 2$\forall\, n : \mathbb{N} \;\bullet\; P(\text{leaf}\;n)$
Inductive step$\forall\, t_1, t_2 : \text{Tree} \;\bullet\; P(t_1) \wedge P(t_2) \;\Rightarrow\; P(\text{branch}(t_1, t_2))$

Formally:

$$\dfrac{P(\text{stalk}) \qquad \forall\, n : \mathbb{N} \;\bullet\; P(\text{leaf}\;n) \qquad \forall\, t_1, t_2 \;\bullet\; P(t_1) \wedge P(t_2) \Rightarrow P(\text{branch}(t_1, t_2))}{\forall\, t : \text{Tree} \;\bullet\; P(t)}$$

Three cases, not two: Unlike sequence induction (which has a single base case ⟨⟩), tree induction has two base cases: stalk and leaf. This is because the free type has two non-recursive constructors. The inductive step covers branch, the only recursive constructor.

Statement to Prove

Task: Prove by structural induction on $\text{Tree}$ that for all trees $t$: $$\#(\text{flatten}\;t) = \text{count}\;t$$

Let $P(t)$ be the predicate:

$$P(t) \;\equiv\; \#(\text{flatten}\;t) = \text{count}\;t$$

We prove $P(t)$ for all trees $t$ by structural induction. This yields three sub-problems:

Part (a) — Base Case 1: $t = \text{stalk}$

Task: Show $P(\text{stalk})$, i.e., show the base case:

$$\#(\text{flatten}\;\text{stalk}) = \text{count}\;\text{stalk}$$

Step 1 — Understand the task

This is the first base case of our induction. We must show that the anti-length-count equation holds when $t$ is stalk. We do this by simplifying both sides using equational reasoning and showing they meet in the middle.

Step 2 — Identify the approach

We will work from the left-hand side. First apply flatten.1 to replace flatten stalk with ⟨⟩, then apply len.1 to get 0, and finally use count.1 backwards to recognise 0 as count stalk.

Step 3 — Plan the components

The chain of reasoning:

  • #(flatten stalk) → flatten.1 → #⟨⟩ → len.1 → 0 → count.1 → count stalk

Three steps, each applying a single definition.

Both sides equal $0$. The base case $P(\text{stalk})$ holds.

Part (c) — Inductive Step: $t = \text{branch}(t_1, t_2)$

Task: Assume $P(t_1)$ and $P(t_2)$ (the inductive hypotheses):

  • IH1: $\;\#(\text{flatten}\;t_1) = \text{count}\;t_1$
  • IH2: $\;\#(\text{flatten}\;t_2) = \text{count}\;t_2$

Show $P(\text{branch}(t_1, t_2))$, i.e., show that:

$$\#(\text{flatten}(\text{branch}(t_1, t_2))) = \text{count}(\text{branch}(t_1, t_2))$$

Vocab for Part (c)

TermDefinition

branch(t1, t2)A tree with left subtree t1 and right subtree t2.
Distributivity of # over concatenationThe key property len.3: #(s ⌢ t) = #s + #t. The length of two sequences joined equals the sum of their individual lengths.

Step 1 — Understand the task

This is the inductive step. We have two inductive hypotheses (one for each subtree), and we must show the property for branch(t1, t2). The crucial ingredient is len.3, which bridges the ⌢ produced by flatten with the $+$ used by count.

Step 2 — Identify the approach

We work from the left-hand side in a single chain:

  1. Apply flatten.3 to expand flatten(branch(t1, t2)) into flatten t1 ⌢ flatten t2.
  2. Apply len.3 to distribute # over ⌢, converting concatenation to addition.
  3. Apply both inductive hypotheses to replace #(flatten ti) with count ti.
  4. Apply count.3 backwards to fold back into count(branch(t1, t2)).

Step 3 — Plan the components

The chain of reasoning:

  • #(flatten(branch(t1, t2))) → flatten.3 → #(flatten t1 ⌢ flatten t2) → len.3 → #(flatten t1) + #(flatten t2) → IH1 ∧ IH2 → count t1 + count t2 → count.3 → count(branch(t1, t2))

Four steps. The len.3 step is the key move — it is what converts the ⌢ structure into the $+$ structure.

Solution

We show the LHS equals the RHS by a chain of equalities, working left to right.

Build Step 1 — Start with the LHS and apply flatten.3

SymbolPlain EnglishWhat it does here
$\text{flatten}(\text{branch}(t_1, t_2)) = \text{flatten}\;t_1 \frown \text{flatten}\;t_2$flatten.3 — flattening a branch concatenates the flattened subtreesexpands the flatten of a branch into a concatenation

Write the left-hand side, substituting $t = \text{branch}(t_1, t_2)$, and apply flatten.3:

$$\#(\text{flatten}(\text{branch}(t_1, t_2))) \;=\; \#(\text{flatten}\;t_1 \;\frown\; \text{flatten}\;t_2)$$

Build Step 2 — Apply len.3 to distribute # over ⌢

SymbolPlain EnglishWhat it does here
$\#(s \frown t) = \#s + \#t$len.3 — length distributes over concatenationconverts the concatenation ⌢ inside # into a sum +

Apply len.3 with $s := \text{flatten}\;t_1$ and $t := \text{flatten}\;t_2$. This is the crucial step — it converts the ⌢ produced by flatten into the $+$ used by count:

$$\#(\text{flatten}\;t_1 \;\frown\; \text{flatten}\;t_2) \;=\; \#(\text{flatten}\;t_1) + \#(\text{flatten}\;t_2)$$

Build Step 3 — Apply both inductive hypotheses (IH1 and IH2)

SymbolPlain EnglishWhat it does here
$\#(\text{flatten}\;t_1) = \text{count}\;t_1$IH1 — the inductive hypothesis for the left subtreereplaces #(flatten t1) with count t1
$\#(\text{flatten}\;t_2) = \text{count}\;t_2$IH2 — the inductive hypothesis for the right subtreereplaces #(flatten t2) with count t2

Apply both inductive hypotheses simultaneously. IH1 replaces $\#(\text{flatten}\;t_1)$ with $\text{count}\;t_1$, and IH2 replaces $\#(\text{flatten}\;t_2)$ with $\text{count}\;t_2$:

$$\#(\text{flatten}\;t_1) + \#(\text{flatten}\;t_2) \;=\; \text{count}\;t_1 + \text{count}\;t_2$$

Build Step 4 — Recognise count t1 + count t2 as count(branch(t1, t2)) using count.3

SymbolPlain EnglishWhat it does here
$\text{count}(\text{branch}(t_1, t_2)) = \text{count}\;t_1 + \text{count}\;t_2$count.3 — counting a branch sums the counts of its subtreesused backwards: count t1 + count t2 = count(branch(t1, t2)), completing the chain to the RHS

Apply count.3 backwards: $\text{count}\;t_1 + \text{count}\;t_2 = \text{count}(\text{branch}(t_1, t_2))$. This completes the chain:

$$\text{count}\;t_1 + \text{count}\;t_2 \;=\; \text{count}(\text{branch}(t_1, t_2))$$

Complete answer

Case 3 chain:

$$\#(\text{flatten}(\text{branch}(t_1, t_2))) \;\underset{\text{flatten.3}}{=}\; \#(\text{flatten}\;t_1 \frown \text{flatten}\;t_2) \;\underset{\text{len.3}}{=}\; \#(\text{flatten}\;t_1) + \#(\text{flatten}\;t_2) \;\underset{\text{IH}}{=}\; \text{count}\;t_1 + \text{count}\;t_2 \;\underset{\text{count.3}}{=}\; \text{count}(\text{branch}(t_1, t_2))$$

Both sides equal $\text{count}\;t_1 + \text{count}\;t_2$. The inductive step is complete.

Summary

PartWhat it provesKey laws used
(a) Base case 1$P(\text{stalk})$: both sides equal $0$flatten.1, len.1, count.1
(b) Base case 2$P(\text{leaf}\;n)$: both sides equal $1$flatten.2, len.2, count.2
(c) Inductive step$P(t_1) \wedge P(t_2) \Rightarrow P(\text{branch}(t_1, t_2))$: both sides equal $\text{count}\;t_1 + \text{count}\;t_2$flatten.3, len.3, IH1 & IH2, count.3

The distributivity of $\#$ is crucial: The entire branch case hinges on len.3: $\#(s \frown t) = \#s + \#t$. Without this property, we could not decompose $\#(\text{flatten}\;t_1 \frown \text{flatten}\;t_2)$ into separate parts matching the recursive structure of count. The len.3 step bridges the ⌢ world (flatten) and the $+$ world (count).

Conclusion

Conclusion: By the induction principle for $\text{Tree}$ (all three obligations discharged), we have:

$$\forall\, t : \text{Tree} \;\bullet\; \#(\text{flatten}\;t) = \text{count}\;t$$

The proof works because count and flatten share the same recursive structure, and $\#$ is a homomorphism from $(\text{seq}\;\mathbb{N},\;\frown)$ to $(\mathbb{N},\;+)$: it maps the concatenation used by flatten into the addition used by count.
