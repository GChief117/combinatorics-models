Exercise 46 — Binary Trees: count and flatten

Structural recursion exercise — defining functions over a free type step by step.

The Scenario

The set of all binary trees with natural number leaves is defined by a free type with three constructors:

$$\text{Tree} ::= \text{stalk} \mid \text{leaf}\langle\!\langle \mathbb{N} \rangle\!\rangle \mid \text{branch}\langle\!\langle \text{Tree} \times \text{Tree} \rangle\!\rangle$$

Every element of Tree is built from exactly one of these three constructors. The definition is exhaustive: there is no other way to build a Tree.

Free Type Definition

Walkthrough — Reading the Free Type

A value of type Tree is built from exactly one of three constructors:

ConstructorArgumentsMeaning
stalk None (a constant) An empty tree — a bare stalk with no data. This is the base case; it carries no natural number and has no children.
leaf⟨⟨N⟩⟩ One natural number $n$ A leaf node holding the value $n$. Written leaf $n$. This is where data lives in the tree.
branch⟨⟨Tree × Tree⟩⟩ A pair of trees $(t_1, t_2)$ A branch node combining two subtrees. Written branch$(t_1, t_2)$. This is the recursive case.

Why "free" type? A free type means every value of type Tree is built by applying constructors a finite number of times, with no hidden equalities. Two trees are equal if and only if they are built from the same constructor with the same arguments.

Structural recursion: To define a function on a free type, we use structural recursion — give one clause for each constructor. Because the cases are exhaustive and the recursive calls are on strictly smaller subtrees, the function is guaranteed to terminate and is total.

Part (a) — Define count : Tree → N

Task: Define a function count that returns the total number of leaves in an element of Tree.

Step 1 — Understand the task

We need to define a function using structural recursion. The function count takes any Tree and returns a natural number: the total number of leaves. Because Tree is a free type with three constructors, we must give one clause for each.

Step 2 — Identify the approach

Every function defined by structural recursion on Tree follows the same skeleton:

  • Base case (stalk): what does count return for the empty tree?
  • Leaf case: what does count return for a single leaf?
  • Recursive case (branch): how do we combine count applied to each subtree?

The three cases are exhaustive, so no other cases are needed.

Step 3 — Plan the components

stalk: a bare stalk has no leaves → return 0.

leaf $n$: a single leaf is exactly one leaf, regardless of the value $n$ → return 1.

branch$(t_1, t_2)$: the leaves of the whole tree are the leaves from $t_1$ plus the leaves from $t_2$ → return count $t_1$ + count $t_2$.

Build Step 4 — branch case

The recursive case: a branch combines two subtrees t_1 and t_2. The total leaf count is count applied to $t_1$ + count applied to $t_2$. The branch node itself is not a leaf, so it does not add 1. Use ∀ to quantify over all pairs of subtrees:

$$\forall\, t_1, t_2 : \text{Tree} \;\bullet\; \text{count}(\text{branch}(t_1, t_2)) = \text{count}\;t_1 + \text{count}\;t_2$$

Complete answer

$$\text{count} : \text{Tree} \to \mathbb{N}$$ $$\text{count}\;\text{stalk} = 0$$ $$\forall\, n : \mathbb{N} \;\bullet\; \text{count}(\text{leaf}\;n) = 1$$ $$\forall\, t_1, t_2 : \text{Tree} \;\bullet\; \text{count}(\text{branch}(t_1, t_2)) = \text{count}\;t_1 + \text{count}\;t_2$$

Task: Define a function flatten that maps a tree to a sequence of natural numbers, preserving the multiplicity of each number that appears.

Step 1 — Understand the task

We need to define a function using structural recursion. The function flatten takes any Tree and returns a sequence of natural numbers: the leaf values collected left-to-right. Because Tree is a free type with three constructors, we must give one clause for each.

Step 2 — Identify the approach

The same structural recursion skeleton applies as for count:

  • Base case (stalk): what sequence does the empty tree produce?
  • Leaf case: what sequence does a single leaf produce?
  • Recursive case (branch): how do we combine the sequences from each subtree?

The key difference from count: flatten collects values, not a count. The combining operation is ⌢ (concatenation) rather than $+$ (addition).

Step 3 — Plan the components

stalk: a bare stalk has no leaves to collect → return the ⟨⟩ empty sequence.

leaf $n$: a single leaf holds value $n$ → return the ⟨n⟩ singleton sequence.

branch$(t_1, t_2)$: the values from the whole tree are the values from $t_1$ followed by the values from $t_2$ → return flatten $t_1$ ⌢ flatten $t_2$.

The recursive case: a branch combines two subtrees t_1 and t_2. Flattening concatenates the sequences from the left subtree and the right subtree. This preserves left-to-right order: all values from $t_1$ appear before all values from $t_2$. Use ∀ to quantify over all pairs of subtrees:

$$\forall\, t_1, t_2 : \text{Tree} \;\bullet\; \text{flatten}(\text{branch}(t_1, t_2)) = \text{flatten}\;t_1 \;\frown\; \text{flatten}\;t_2$$

Summary

Comparing the two functions: Both follow the same structural recursion dictated by the free type. They differ only in the codomain and the combining operation:

Casecount (to $\mathbb{N}$)flatten (to seq $\mathbb{N}$)
stalk$0$$\langle\rangle$
leaf $n$$1$$\langle n \rangle$
branch$(t_1, t_2)$count $t_1$ $+$ count $t_2$flatten $t_1$ $\frown$ flatten $t_2$

In both cases, stalk maps to the identity element ($0$ for addition, $\langle\rangle$ for concatenation), leaf maps to a unit ($1$ or a singleton), and branch combines results with the appropriate binary operation ($+$ or $\frown$).

Totality: Because Tree is a free type, every tree is built from a finite number of constructor applications. The three cases are exhaustive and the recursive calls are on strictly smaller subtrees. This guarantees that both count and flatten are total functions.
